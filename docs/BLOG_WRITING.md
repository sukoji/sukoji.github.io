# 블로그 글 작성 가이드

에이전트·본인 모두 이 문서를 참고해 `_posts/` 글의 톤과 포맷을 맞춘다.

**템플릿:** [`_posts/_TEMPLATE.md`](../_posts/_TEMPLATE.md)  
**예시 글:** [`_posts/2026-07-10-gpt-5-6-first-impressions.md`](../_posts/2026-07-10-gpt-5-6-first-impressions.md)

---

## 1. 파일 규칙

```
_posts/YYYY-MM-DD-slug.md
```

### Front matter (필수)

```yaml
---
layout: post
title: "구체적인 제목"
date: 2026-07-10
tags: [주제, 하위주제]
summary: 목록·SNS에 노출되는 한 줄 요약
---
```

| 필드 | 규칙 |
|------|------|
| `title` | 검색 가능하게. "메모", "정리", "나온 날" 같은 빈 제목·뉴스 레이아웃 지양 |
| `summary` | 1문장, 80자 내외 |
| `tags` | 2~4개, 소문자·하이픈 |

---

## 2. 톤 (AI 티 줄이기)

### 하지 말 것

- "혁신적인", "획기적인", "놀라운", "완벽한" 같은 마케팅 수식어
- 모든 섹션을 똑같은 길이·구조로 맞추기
- 다른 블로그/보도자료 문장 **복붙** (출처 링크 + 자기 말로 요약만)
- 불필요한 이모지, 과한 bullet 남발
- "결론적으로", "요약하자면"으로 매 섹션 끝내기

### 할 것

- **짧은 문단** (2~4문장)
- **1인칭 경험** — "돌려봤다", "예정이다", "sympo에서 봤다"
- **불확실성 표현** — "아직", "직접 써봐야", "~일 듯"
- **구체적 고유명사** — 모델명, repo명, 논문명, 숫자
- 마무리에 **다음 액션** 하나 (실험, 로그, 읽을 논문)

---

## 3. 본문 구조 (권장)

```
1. 리드 (2~3문장) — 사건·배경
2. callout-summary — 한 줄 요약
3. 본문 섹션 (h2) — 2~4개
4. (선택) 명령어 / 코드 블록
5. (선택) callout-warn — 주의·한계
6. 참고 링크
```

섹션 제목은 명사형 또는 질문형. "소개", "배경" 같은 빈 제목 지양.

---

## 4. 시각·정보 계층 (마크다운)

### 한 줄 요약 박스

```markdown
> **한 줄 요약**
> 핵심 결론 한 문장.
{: .callout-summary}
```

### 주의 박스

```markdown
> **주의**
> 실패 모드, 비용, 제한.
{: .callout-warn}
```

### 표 — 스펙·비교·가격

```markdown
| 모델 | 용도 | 가격 |
|------|------|------|
| **Sol** | ... | ... |
```

### 강조

- `**굵게**` — 모델명, 결론, 숫자 강조
- `` `코드` `` — API명, 파일명, 플래그
- 일반 링크 — 공식 문서·본인 repo 우선

### 명령어 블록 — **복붙 가능하게**

```markdown
```bash
# 용도 한 줄 주석
curl ... -d '{ ... }'
```
```

- 실제 동작하는 형태로 작성
- `$` 프롬프트는 넣지 않음 (복붙 방해)
- 긴 JSON은 최소 예시만

### 이미지

```markdown
![설명](/assets/images/posts/2026-07-10-slug.png)
```

- 스크린샷·다이어그램은 `assets/images/posts/`에 저장
- alt 텍스트 필수
- repo README 이미지 hotlink 가능하나, 오래 쓸 그림은 로컬 복사 권장

## 5. 글자 색·스타일 (사이트 CSS 기준)

| 요소 | 렌더링 |
|------|--------|
| 본문 | 회색 톤 (`--ink-soft`) |
| **굵은 글** | 거의 검정 (`--ink`) |
| 링크 | teal (`--theme`) |
| `코드` | 연한 회색 배경 + 녹색 계열 글자 |
| 코드 블록 | 어두운 배경, 밝은 글자 |
| callout-summary | 연한 녹색 배경 |
| callout-warn | 연한 주황 배경 |

색은 마크다운 클래스로만 제어. HTML `<span style>` 사용 금지.

## 6. 참고 링크 섹션

마지막에 `## 참고` 로 모음. 형식:

```markdown
## 참고

- [OpenAI — GPT-5.6](https://openai.com/index/gpt-5-6/)
- [본인 repo — sympo](https://github.com/sukoji/sympo)
```

## 7. 주제별 체크리스트 (에이전트용)

작성 후 확인:

- [ ] `summary`가 front matter에 있음
- [ ] 첫 문단만 읽어도 주제가 이해됨
- [ ] callout-summary 1개 이상
- [ ] 표 또는 코드 블록 중 하나 이상 (기술 글일 때)
- [ ] 다른 글에서 문장 베끼지 않았음
- [ ] 참고 링크에 1차 출처 포함
- [ ] 과장·마케팅 문구 없음
- [ ] `bundle exec jekyll serve`로 로컬 미리보기 확인
